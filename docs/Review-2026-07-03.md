## API (High)

**H1. 401/403/500 responses violate the RFC 9457 contract.** `src/SraRms.Api/Program.cs:35,72-84` — `AddProblemDetails()` is registered but `app.UseExceptionHandler()` / `app.UseStatusCodePages()` are not added. Auth-middleware 401/403s are empty-bodied and Production 500s have no problem+json body, while the contract promises `application/problem+json` on every error. Fix: add both middlewares after `builder.Build()`.

**H2. Shrinking a project's date window strands allocations outside it.** `src/SraRms.Api/Controllers/ProjectsController.cs:120-141` — `Update` validates `endDate >= startDate` but never re-checks existing allocations against the new window (SRS §3.5, FR-ALL-5; UC-5 makes end-date edits a primary flow). Fix: reject with 409/400 or surface a non-blocking `warnings` array mirroring the allocation approach.

## API (Medium)

**M1. Check-then-insert uniqueness races → raw 500s instead of 409.** All duplicate checks (`ClientsController.cs:45`, `ResourcesController.cs:56,125`, `ProjectsController.cs:62,126`, `ReferenceDataController.cs:43`) are `AnyAsync` + `SaveChangesAsync` with no `DbUpdateException` handling; the DB's unique indexes will throw on concurrent duplicates. Fix: catch Postgres `23505` and map to the existing 409 problem.

**M2. Uploaded profile images served with no authentication.** `Program.cs:80` + `ResourcesController.cs:175-182` — `UseStaticFiles()` runs before auth and serves `wwwroot/uploads/resources/*` anonymously; NFR-SEC-5 requires personal data (incl. images) to be access-controlled. Also: no upload size cap, and replacing a png with a jpg orphans the old file. Fix: serve via an authorized `GET /v1/resources/{id}/image` endpoint; cap and sniff uploads.

**M3. Gantt "projects" view hardcodes `overAllocated = false`.** `DashboardController.cs:114` — presents an affirmative wrong answer (FR-GANTT-3); the resources view computes it properly. Fix: set it to `null` (suppressed by `WhenWritingNull`) or compute from the bar resource's overlapping allocations.

**M4. Dashboard and utilisation report load entire tables into memory.** `DashboardController.cs:28-29`, `ReportsController.cs:25-27` — `Include(r => r.Allocations).ToListAsync()` materialises every resource with all historical allocations per request; won't meet NFR-PERF-1 at the 10k-resource NFR-SCALE-1 target. Fix: filtered includes / server-side aggregation for the requested window.

**M5. CSV formula injection in report exports.** `ReportsController.cs:144-149` — `Escape` doesn't neutralise cells starting `=`, `+`, `-`, `@`; a client named `=HYPERLINK(...)` executes when finance opens the CSV (UC-4). Fix: prefix such cells with `'`.

**M6. Test factory mutates process-wide env state.** `tests/SraRms.Api.Tests/ApiFixture.cs:88` — `Environment.SetEnvironmentVariable` is global and never cleaned up; breaks the moment a second factory or parallel collection appears. Fix: `ConfigureAppConfiguration` + `AddInMemoryCollection` inside `ConfigureWebHost`.

**M7. Test gaps beyond TODO.md's list.** Untested: all Project endpoints (duplicate-code 409, bad dates 400, delete-with-dependents 409/cascade), allocation PUT/DELETE and top-level POST, problem+json content-type on any 4xx, paging/sorting semantics (`pageSize` clamping, `-desc` prefix), CSV escaping, allocation/budget reports, image upload. Suggested first additions: a `ProjectsTests` class mirroring `ClientsTests` + one problem+json shape assertion in `IntegrationTestBase`.

## API (Low)

- **L1.** Nested list routes reset out-of-range `pageSize` to 25 while top-level routes clamp to 200 (`ClientsController.cs:142` vs `Contracts/Common.cs:37`). Extract one clamping helper.
- **L2.** `format=xml` silently returns JSON (`ReportsController.cs:121`); contract enum is `[json, csv]` → return 400.
- **L3.** Missing project on top-level allocation create → 400, nested route → 404 (`AllocationsController.cs:52` vs `ProjectsController.cs:191`). Pick one (404 reads best).
- **L4.** `q` interpolated into ILIKE unescaped — `%`/`_` act as wildcards (`ClientsController.cs:21`). Escape if exact-substring semantics matter.
- **L5.** Unknown `sort` fields silently fall back to default order; consider 400.
- **L6.** `resourceStatuses` reference items all share `id: Guid.Empty` (`ReferenceDataController.cs:21`); use deterministic ids.
- **L7.** PUT /projects with omitted `remaining` nulls it, and the budget report coerces null→0 ("100% consumed") (`ProjectsController.cs:138`, `ReportsController.cs:101-103`). Default on update or flag null in the report.

## Web front end

**High**

**W-H1. Timezone off-by-one in default report range.** `web/src/views/ReportsView.vue:12-15` — `toISOString()` converts local midnight to UTC; in Australia/Sydney the July "From" default becomes 30 June, silently shifting utilisation by a day. Fix: format dates locally.

**W-H2. Field-level validation errors discarded.** `web/src/api/http.ts:31-35` — only `problem.detail || problem.title` is surfaced; the RFC 9457 `errors` dictionary is never shown, so multi-field 400s collapse to one generic toast. Fix: render `e.problem?.errors` per field.

**Medium**

- **W-M1. Keyboard accessibility fails WCAG 2.1 AA (NFR-USE-2).** Mouse-only clickable rows (no `tabindex`/keydown) across all list/dashboard views; `ModalDialog.vue` lacks Escape handling, focus trap, and `aria-labelledby`; labels unbound to inputs; unnamed ✕ button; `href="#"` anchors in `AllocationsView.vue:73`; double `h1` per page. Fix: RouterLinks in first cells, modal focus management, `label for=`, demote topbar heading.
- **W-M2. SRS gaps missing from TODO.md:** dashboard `from`/`to` horizon never passed (FR-DASH-2), stat cards not drill-down links (FR-DASH-3), no project/resource filter UI on allocations (FR-ALL-2), allocation + budget reports have no client/UI at all (FR-REP-2/3), no image-upload UI (FR-RES-5), utilisation `department` filter typed but no control. Add to backlog.
- **W-M3. CSV export URL fragile.** `web/src/api/index.ts:74-75` — no fallback when `VITE_API_BASE` unset, params unencoded, omits `department`, and a plain `<a href>` can't carry the planned MSAL bearer token. Fix: axios blob download.
- **W-M4. Over-allocation warnings shown as errors, first-only.** `ProjectDetailView.vue:55` — non-blocking FR-ALL-6 warnings rendered in red as if the create failed. Fix: add a `warning` toast kind; show all, prefixed "Allocation created —".

**Low**

- No catch-all 404 route (`router/index.ts`); detail views blank on load failure (no `v-else` error state); TS types stricter than contract required-lists (`types/index.ts:25-26,44,96-98`); load/debounce/pager scaffolding copy-pasted across four views (extract `usePagedList` composable); dropdowns hard-capped at 200 items with no truncation warning (`ProjectsView.vue:48`).

## Database

- **D-M1. No migration runner; V001 not safely re-runnable.** `db/migrations/V001__initial_schema.sql:212-217` — hand-applied via psql; `schema_migration` insert is unconditional and nothing checks it before applying. Naming is already Flyway-compatible → adopt Flyway/DbUp, or at minimum `ON CONFLICT DO NOTHING` + a guard script.
- **D-M2. Wrong index shape for overlap queries.** `V001:167,194` — composite b-tree `(start_date, end_date)` only exploits the leading column for `start <= :to AND end >= :from`. Fix: GiST index on `daterange(start_date, end_date, '[]')`, query with `&&`.
- **D-L1.** No `remaining <= budget` check (`V001:154`).
- **D-L2.** Allocation-within-project-window rule is app-side only (deliberate, but a constraint trigger would guard non-API writers).
- **D-L3.** `client.name` and `resource.email` lack trgm indexes despite the UI advertising substring search.
- **D-L4.** `docs/openapi.yaml:22` says local dev is port 5000; everything else uses 5163. Update the servers entry.

## Done well

- Faithful contract implementation: every path/verb matches `openapi.yaml`, correct envelopes, Location headers, role split per operation, and over-allocation implemented exactly as a non-blocking `warnings` array (FR-ALL-6).
- Defence in depth: DB mirrors the API invariants (case-insensitive uniques, date CHECKs, RESTRICT FKs); integration tests run the real migration in Testcontainers Postgres.
- Enum tokens byte-identical across Postgres, TS, and OpenAPI — a whole class of drift bugs eliminated.
- Clean auth composition: deny-by-default `[Authorize]`, per-action policies, Dev bypass double-gated, dev secrets git-ignored, audit attribution centralised in `SaveChanges`.
- One well-designed error spine in the SPA (axios interceptor → typed `ApiError`), consistent loading/empty states, brand tokens sourced from the actual guidelines (NFR-USE-1).
